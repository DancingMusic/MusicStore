登录模型

SDK 只提供一个可选入口：login(request)。扫码、OAuth、网页弹窗、设备码和手动 token 都是连接器自己的实现细节，宿主只渲染返回的 actions；桌面端可按 cookieCapture 元数据打开官方网页并回传 cookie。

1status

宿主先询问当前状态，连接器返回 anonymous、authenticated、expired 或 error。

2start

用户点击登录后，连接器返回二维码、open-url、manual-input 或 message。

3continue

QR/设备码流程用 flowId 继续轮询；网页登录可把桌面端读取到的 cookie 放进 input 传回连接器。

4configPatch

成功后只持久化连接器返回的 cookie/token 补丁，不在宿主增加平台字段。

const result = await connector.login({ intent: "start" });

for (const action of result.actions ?? []) {
  if (action.type === "qr") renderQr(action.imageUrl ?? action.qrUrl);
  if (action.type === "open-url" && action.cookieCapture) {
    const cookie = await openDesktopLoginWindow(action.cookieCapture);
    await connector.login({ intent: "continue", flowId: result.flowId, input: { cookie } });
  } else if (action.type === "open-url") {
    openInPlayerModal(action.url);
  }
  if (action.type === "manual-input") renderForm(action.fields);
}

if (result.configPatch) {
  persistConnectorConfig(connector.meta.id, result.configPatch);
}
网页登录必须优先在播放器内承载。桌面端可用独立官方登录窗口读取允许的 cookie；Web demo 无法跨站读取 cookie 时，只展示 iframe/外部打开兜底。

连接器契约

MusicConnect-* 实现仓库只依赖 MusicStore SDK 类型，不直接依赖 DancingMusic 宿主。每个实现都必须发布 ESM bundle、契约测试和 GitHub Pages demo。

统一类型

实现 MusicConnector，返回带平台前缀的 MusicTrack.id。

声明高级配置

需要代理、cookie 或 token 时，通过 meta.configSchema 声明字段；宿主应把它放在高级设置里，普通用户优先走登录入口。

真实可播放

默认激活连接器必须返回真实 stream，禁止 mock 或点播后才失败。

Capability含义宿主行为
search搜索曲目展示搜索结果和分页。
stream解析播放地址点击播放时调用 getStreamUrl(trackId)。
lyrics原生歌词优先使用当前连接器；否则可走宿主跨连接器 fallback。
playlist精选歌单展示歌单列表和歌单曲目。
login账号登录调用唯一入口 login(request) 并渲染 actions。

宿主接入

DancingMusic 不打包任何连接器实现。默认清单保存仓库、tag 和 bundle URL；运行时通过动态 ESM import 加载。

const record = {
  id: "netease-cloud-music",
  source: "https://github.com/DancingMusic/MusicConnect-NetEase",
  bundleUrl: "https://cdn.jsdelivr.net/gh/DancingMusic/MusicConnect-NetEase@v0.5.0/dist/index.js",
};

const mod = await import(record.bundleUrl);
const connector = new mod.default();
await connector.init(savedConnectorConfig);

发布检查

连接器发布前先保证契约测试、bundle、demo 页面和配置说明同步，避免宿主加载到不可用实现。

需要代理/token 的数据源必须把字段放在高级设置，并明示缺少数据代理时的影响。
